# AdobeのCreative Cloudへの移行と買い切りからサブスクへの転換

## 俯瞰（Overview）

Photoshop、Illustrator、Premiere Pro、After Effects——Adobeが持つソフトウェアは「クリエイターの業界標準」という特殊な地位を持つ。デザイナーが求人に応募する時、「Photoshopが使えます」と書く。映像プロダクションがフリーランスを採用する時、「Premiere Proで納品」を前提にする。Adobe製品を知らないと、クリエイティブ業界に入れない——これが2013年のAdobeが持っていた最大の武器だった。

この武器を使って、Adobeは業界で最もリスクの高いビジネスモデル転換の一つを実行した。永続ライセンス（買い切り）の廃止とサブスクリプション一本化だ。Creative Suiteの最後のバージョンはCS6（2012年）。CS7は存在しない。「新機能が欲しければCreative Cloudに入るしかない」という構造的な強制だった。

批判は激しかった。Adobeのフォーラムは「Adobeをボイコットする」という投稿で溢れた。しかし2年後、批判した人々のほとんどはCreative Cloudに加入していた——なぜなら「Adobe製品を使わない」という選択は、業界から退場することを意味したからだ。

この移行が産業史に残る理由は、「強制転換が成功した条件」を最も明確に示した事例だからだ。その条件とは「スイッチングコストが十分に高く、競合が業界標準を奪えない」という状況だ。

## 構造分析

### 何が起きていたか

**2007-2012：サブスク移行の準備期間**

Adobeのビジネスモデルには長年の課題があった。「バージョンアップのたびに大型支出が発生する」という顧客側の問題と、「製品サイクルに売上が連動する（バージョンアップ年は好決算、翌年は低調）」という業績の不安定性だ。

2007年、AdobeはPhotoshop.comという月額サービスを試験的に開始した（2012年に終了）。2011年には「Creative Cloud」という名称でサブスク版を永続ライセンスと並行して提供し始めた。この「選択肢として提供」フェーズで、どの顧客層がサブスクを選ぶかのデータを収集した。

一方、Adobeのビジネス上の本当の問題は「不法コピー」だった。Photoshopは世界で最もコピーされているソフトウェアの一つで、特に新興国市場では正規ライセンスユーザーより不正コピーユーザーの方が多かった可能性がある。クラウドベースのサブスクに移行すれば、ローカルインストールが前提の不正コピーを構造的に防げる。

**2013年5月：「CS7は存在しない」宣言**

Adobe MAX 2013（クリエイターカンファレンス）の前週、AdobeはCS7（Creative Suite 7）を出さないと発表した。今後の新機能はCreative Cloudのみで提供される——この「選択肢の撤廃」が転換点だった。

発表直後のTwitter・フォーラムの反応は否定的だった。「Adobeは終わった」「スイッチする」という声が相次いだ。英国議会でも「消費者の権利を侵害する」として議論になった。しかし代替手段は実質的に存在しなかった。Affinity Designerはまだ存在せず（2014年にβ版）、GIMPはプロには力不足、Sketch（2010年）はMacのみで機能も限定的だった。

**2014-2016：批判から加入へ——フライホイールの始動**

移行後2年間の数字が戦略の正しさを証明した。2014年のCreative Cloud加入者は380万人（2013年の約3倍）。2016年には1100万人を超え、Adobeの売上は移行前の年間18億ドルから35億ドルへと倍増した。

重要なのは「加入者あたり単価」の変化だ。Creative Suite 6の永続ライセンスは約$2600だった。これを3-4年ごとにアップグレードするとすると、年換算で$650-870程度。Creative Cloudの月額は当初$50（年額$600）だった。数字上は「ほぼ同等」——しかしAdobeにとっては「必ず毎年入る収益」になった。さらに加入者は解約すると作業ファイルにアクセスできなくなるという事実が、継続加入の強力なインセンティブを作った。

**2017年以降：エコシステムの拡張とAI統合**

安定したARR基盤を持ったAdobeは、Machine LearningのAdobe Sensei（2016年）、株式素材のAdobe Stock、フォントのAdobe Fonts、WebデザインのAdobe XD、コラボレーションのFrame.ioを次々と買収・開発してCreative Cloudのエコシステムを拡張した。2021年にはFigmaを$200億で買収しようとした（EU規制当局の反対で2023年に断念）。

### なぜそうなったか（力学）

**「業界標準」というロックインの二重構造**:
Adobeのスイッチングコストは二層に分かれている。第一層は「ファイル互換性」——.psd、.ai、.aep というAdobe固有のファイル形式は、Adobe以外のソフトで完全に開けない（開けても崩れる）。10年分の制作物が全部Adobe形式で保存されている。第二層は「スキルの移植コスト」——Photoshopで10年培ったワークフローは、Affinity Photoに移ると一から学び直しになる。この二重のロックインが「嫌なら使うな」という宣言を可能にした。

**「選択肢を先に増やし、後で撤廃する」段階設計**:
2011-2012年の「永続ライセンスとCC並行提供」は単なる試験ではなく、「サブスクに慣れさせる」段階だった。既存ユーザーにCCの価値（クラウドストレージ、常時最新バージョン、Behance連携）を体験させ、「CC便利だな」と思い始めた頃に永続ライセンスを廃止する——移行の摩擦を最小化する設計だった。

**不法コピーの構造的解決**:
サブスクへの移行は、不正コピーを「技術的に無意味」にした。CCはオンライン認証が必要で、インターネット接続なしでは長期使用できない（30日のオフライン使用は可能だが）。不正コピーで使っていたユーザーに「月$10の学生プラン」というオプションを提供したことで、新興国の学生ユーザーを正規顧客に転換する経路を作った。

**「解約すると使えなくなる」という心理的ロック**:
永続ライセンスは「一度買えば一生使える」という安心感を与えていた。CCは「解約した瞬間に.psdファイルを開けなくなる」という恐怖を生む（正確にはファイルはローカルに残るが、Creative Cloudアプリで開けなくなる）。この「失う恐怖」が解約率（チャーン）を構造的に低く保つ最大の要因だった。

### 構造図

```
移行前（2012年まで）:
  [Adobe] ──パッケージ販売─→ [顧客] $2600/3-4年ごと
  → 売上が製品サイクルに連動・不規則
  → 不正コピーを防ぐ手段なし
  → バージョンアップ採用率が低い（古いバージョンを使い続ける顧客多数）

移行後（2013年〜）:
  [Adobe] ──月額$20-55─→ [顧客] × 1100万人以上 = 安定ARR
  ↑
  ロックインの二重構造:
    1. ファイル形式ロックイン（.psd/.ai/.aep ≠ 他社互換）
    2. スキルロックイン（Photoshopの操作習熟）
    3. 作業フローロックイン（Cloud Storage / Fonts / Stock）
  ↑
  解約コスト:
    → 解約=ソフト使用不可（既存ファイルへのアクセス喪失の恐怖）
    → 業界標準を離れる=採用・受注機会の減少

[Affinity / Sketch / Figma 等の競合]
    → 機能・価格で有利だが「業界標準」の地位なし
    → 特定用途（UI設計）では Figma が侵食に成功（→$200億買収提案）
```

## 転換点の意味

### TP1: 2011年 Creative Cloud並行提供——「選択肢として試される」

永続ライセンスとCCを並行提供したフェーズは、AdobeがCCの市場受容性を検証する実験だった。この時期に「CCを選んだ顧客」の解約率・利用頻度・LTVのデータを集積し、強制移行の判断を支える根拠を作った。

なぜ転換点か: 並行提供期間がなければ、2013年の強制廃止は「テストなしの大博打」になっていた。「選択肢として試し、データを見てから廃止」という段階設計が、移行リスクを大幅に低下させた。この「先に選択肢を与えて慣れさせ、後で選択肢を撤廃する」というパターンは、後の多くのSaaS企業の移行戦略のモデルになった。

### TP2: 2013年5月 CS7廃止宣言——「強制転換」の実行

Adobeが「CS7は出ない」と発表した時点で、移行は「不可逆」になった。これはビジネスモデル転換における最も危険な瞬間——既存ユーザーを怒らせるリスクを取りながら、「競合に逃げる顧客が少ない」という賭けに出た。

なぜ転換点か: この瞬間に二つのことが証明された。①業界標準の地位は「嫌なら使うな」を言えるほど強固だった。②「選択肢がない」と悟った顧客の批判は時間とともに沈静化する。この判断の正しさは2年後の数字で証明され、他のソフトウェア企業の永続→サブスク移行の先例となった。

### TP3: 2021年 Figma $200億買収提案——「次のプラットフォーム転換」への対応

FigmaはAdobeにとって「Adobeがまだ支配していない領域」——UIデザイン、コラボレーション、ブラウザベースの設計ツール——で急速に成長していた。$200億という価格は、Figmaの当時の売上（約$4億）の50倍。これはFigmaの業績への対価ではなく「次の業界標準になる前に買収する」戦略的プレミアムだった。

なぜ転換点か: Adobeが$200億を提示した事実は、「Creative Cloudの壁がFigmaには効かない」と認識していたことを示している。Figmaはブラウザベースで、Figmaファイルを使えばAdobeとのスキルロックインが消える（ブラウザなら誰でも使える）。買収はEUの競合保護規制で2023年に断念されたが、「Adobeの次の脅威」を最も明確に示した出来事だった。

## 反対と執念

2013年5月の発表後、Adobeのフォーラムは「怒り」で溢れた。英国議会では「消費者の権利を守れ」という動議が出た。Adobeの株価は発表後3日で7%下落した。しかしAdobeのCEOシャンタヌ・ナラエンは揺るがなかった——「私たちは正しい判断をしている。数年後に証明される」と繰り返した。

内部にも抵抗はあった。「中小規模の顧客を失う」「学生・フリーランスが離れる」という懸念に対し、Adobeは「学生プラン月$20」「フォトプラン月$10（PhotoshopとLightroomのみ）」という低価格セグメントを設けて対応した。この価格設計が、「Adobe高すぎる」という批判を吸収する緩衝材になった。

## 考察・私見

AdobeのCC移行が成功した根本的な理由は「競合他社が業界標準を奪えなかった」ことだ。もしFigmaが2013年に既に完成度が高く、UI/UX以外のクリエイティブ全域をカバーしていたら、CCへの強制移行は顧客流出を招いた可能性がある。業界標準の地位は「あるかないかの二値」ではなく、「どの市場セグメントでどれだけ支配しているか」の問題——FigmaはUI/UXというセグメントでその地位を取った。

CC移行から学べる反直感的な洞察は「顧客の怒りは時間とともに薄れるが、スイッチングコストは薄れない」ということだ。批判したユーザーは半年後にはほぼ全員CCに加入していた——怒りより、業界標準を手放す恐怖が大きかったから。感情と経済合理性が対立した時、長期的には経済合理性が勝つ。

Netflixのストリーミング移行（2011年のQwikster失敗は別として）、MicrosoftのOffice 365、AutodeskのAutoCADサブスク移行——いずれもAdobeのCC移行パターンを参照している。業界の「強制サブスク転換」のプレイブックを書いたという意味で、この事例は2010年代のSaaS移行史の基礎文献だ。

## 未解決の問い

- 2013年時点でAdobeの経営陣は「何%の顧客が残り、何%が競合に逃げる」と予測していたのか。実際の数字はその予測を上回ったのか下回ったのか
- FigmaのEUによる買収阻止（2023年）は、競争促進の観点で正しかったか。Adobeが買収していれば、CreativeとUI/UXの垂直統合で何が変わったか
- 「業界標準でなければ」CCへの強制移行は失敗していたか——業界標準の地位がどのくらい強固であれば同じことが可能か
- Adobe Senseiに始まるAI統合と生成AI（Firefly）は、「Adobe製品を使えば必ずAIが付いてくる」という新しいロックインを作るか
- AdobeがFigmaを$200億で評価した根拠——Figmaの真の脅威はAdobeの既存製品への代替か、それとも「AIネイティブ設計ツール」として次世代の業界標準になる可能性か
