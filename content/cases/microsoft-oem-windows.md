# MicrosoftのOEM戦略とWindowsの世界標準化

## 俯瞰（Overview）

Appleがコンピュータを「ハードウェアとソフトウェアを一体で売るもの」と定義した時、MicrosoftはコンピュータをOSの流通チャネルと定義した。この根本的な違いが、1990年代のPC産業の構造を決定した。

ビル・ゲイツが1980年にIBMと結んだ契約は、一見するとIBMへのOS供給という単純な取引に見えた。しかし実態は「IBMがPCを売れば売るほど、Microsoftへのロイヤリティが積み上がる」という構造だった。さらにIBMがPCアーキテクチャをオープンにしたため、DellやCompaqやHPがIBM互換PCを作り始めた。それらすべてにMicrosoftはライセンスを求めることができた——IBMは自らのオープン戦略でMicrosoftへの依存度を高めた。

Microsoftの戦略の本質は「ネットワーク効果の地政学的独占」だ。Windowsを使うユーザーが多いほど、Windowsで動くアプリが多く作られる。アプリが多いほど、ユーザーはWindowsを選ぶ。Microsoftはこの正のフィードバックループを、OEMプレインストールという流通チャネルで物理的に確保した。PCを買った瞬間からWindowsが入っている——この「デフォルト」であることが、競合OSが入り込む余地を消した。

## 構造分析

### 何が起きていたか

**1980-1981：「ライセンスを保持する」という一つの決断**

IBMが秘密裡に進めていたPCプロジェクト（Project Chess）は、開発スピードを上げるために外部調達を決めた。CPU（Intel 8088）、OS（Microsoft）、BASIC（Microsoft）。IBMのエンジニアがゲイツに会いに来た時、Microsoftはまだ本格的なOSを持っていなかった。ゲイツはシアトル・コンピュータ・プロダクツから86-DOS（QDOS）を5万ドルで購入し、IBM向けにMS-DOSとして提供した。

決定的な判断は「OSをIBMに売らなかった」ことだ。IBMはOSを買い取ろうとしたが、Microsoftはライセンス（使用権）の供与にとどめた。IBMにとってOSは「部品の一つ」に過ぎず、買い取りに固執しなかった。しかしこの「所有権の保持」が、その後20年のMicrosoftの命運を決めた。

IBMのPCアーキテクチャがオープンになった後、コンパック、ヒューレット・パッカード、デル、AST……無数のIBM互換機メーカーが生まれた。それらすべてにMicrosoftはMS-DOSのライセンスを求めることができた。IBMという「最初の顧客」は、事実上Microsoftに世界中のPCメーカーへのアクセスを与えてしまっていた。

**1985-1993：Windowsの離陸と事実標準の確立**

Windows 1.0（1985年）と2.0（1987年）は失敗作だった。しかしWindows 3.0（1990年）はGUIの完成度が上がり、爆発的に普及した。発売後6ヶ月で100万本を突破——当時のソフトウェア販売記録を塗り替えた。

この時期のMicrosoftのOEM戦略で特徴的だったのは「PCメーカーにWindowsをプレインストールする対価として、販売台数連動のロイヤリティを請求する」という仕組みだった。さらにPCメーカーが「WindowsをインストールしないPC」を販売した場合でも、そのPCの台数分のロイヤリティを支払わせるという「per-processor契約」を締結した。結果として、PCメーカーにとってはWindowsを入れない選択肢が経済的に不利になった。DOS/Vや Linux、IBMのOS/2など競合OSは、この「デフォルトの壁」に阻まれた。

**1995-1998：Windows 95とIEバンドリング——独占の拡張**

Windows 95（1995年8月）の発売は、PC産業史上最大のマーケティングイベントの一つとなった。Rolling Stonesの「Start Me Up」を使ったCM、ニューヨーク・タイムズ・スクエアの大型広告、世界同時発売。Windows 95はすでに強固なプラットフォームを持っていたMicrosoftが、さらに「ユーザーエクスペリエンス」という次元でも競合を圧倒した。

しかし競争上の問題が起きたのは、Internet Explorer（IE）の「バンドリング」だった。当時ブラウザ市場はNetscape Navigatorが支配していた。MicrosoftはIEをWindowsに無料でバンドルし、PCメーカーにIEをデスクトップに表示させる義務を課し、NetscapeのインストールをOEMが妨げるよう圧力をかけた。Netscapeの創業者マーク・アンドリーセンは後に「これは不公正な競争だった」と証言している。

**1998-2001：DOJ独禁法訴訟——帝国の法的限界**

1998年5月、米国司法省（DOJ）と20の州政府がMicrosoftを反トラスト法（シャーマン法）違反で提訴した。判事ジャクソンは2000年に「Microsoftはモノポリーを保有しており、それを維持するために競争を制限した」と認定し、Microsoftの会社分割（OSと他ソフトの分割）を命じた。MicrosoftはControl Alt Delete記者会見を開くほど激しく抵抗した。最終的に控訴審は分割命令を破棄し、2001年の和解でMicrosoftはOEM契約を一部変更することで決着した。

### なぜそうなったか（力学）

**「デフォルトの壁」の経済学**:
PCメーカーがWindowsをプレインストールする理由は、「Windowsを入れないと売れない」からだ。消費者はWindowsに慣れており、WindowsのソフトウェアはWindows上でしか動かない。この「アプリの蓄積＝プラットフォームの価値」というネットワーク効果が、Microsoftの競争優位の核心だった。後発のOSがどれだけ高性能でも、「Excelが動かない」「Photoshopが動かない」という一点で選ばれない。

**IBMのオープン戦略という贈り物**:
IBMがPCアーキテクチャをオープンにした理由は「クローンを防ぐ力がない」からではなく、「早期に市場を作るためにオープン化した方が得」と判断したからだ。しかしこの判断の副作用として、PC業界の標準化が進み、差別化がなくなってきた。ハードウェアはコモディティ化し、利益はOSとプロセッサに集中した——MicrosoftとIntelという、Wintelデュオポリーが誕生した。

**「per-processor契約」の威力**:
OEMに「出荷台数ベースのロイヤリティ」を課す方式は、Windowsを入れないPCでもライセンス料が発生するという構造を作った。これは後にFTC（連邦取引委員会）の調査対象になったが、1994年には合意解除となった。それ以降MicrosoftはPC販売台数ではなくWindows販売台数ベースの契約に変更したが、「デフォルトインストール」という慣行は変わらなかった。

**バンドリングという次の武器**:
Windowsが事実標準になると、Microsoftは隣接するソフトウェア市場（ブラウザ、メディアプレイヤー、メーラー、スプレッドシート）を「バンドル」で取り込むことができた。IEのバンドルはNetscapeを潰した最も有名な例。この「OS独占から隣接市場へのてこ入れ」がDOJの独禁法訴訟の中核的主張だった。

### 構造図

```
[Microsoft]
    保持: Windows OS所有権（IBMに売らず、ライセンスのみ）
    収入: OEMライセンス料（PCメーカー全社）
    ↓ Windowsをプレインストールする条件でライセンス提供
[Dell / HP / Compaq / 東芝 / NEC 等（OEM各社）]
    ← 販売台数×ライセンス料をMicrosoftに支払う
    → 「Windows入り」PCを販売することが事実上の市場参加条件
    → IEをデスクトップに表示する義務（1995年以降）
[小売・法人チャネル]
    ↓ 「WindowsのPC」を購入
[最終消費者]
    ← 「Windowsだからアプリが使える」という選択の理由

[Intel CPU]   ←→ [Windows OS]   = Wintelデュオポリー
    ↑ハードウェア標準    ↑ソフトウェア標準

[Netscape / OS/2 / Linux（競合）]
    ← デフォルト採用なし → 市場シェア取れず
```

## 転換点の意味

### TP1: 1981年 IBM PCへのMS-DOSライセンス供給——「OS所有権の保持」

ビル・ゲイツとポール・アレンがシアトル・コンピュータ・プロダクツから5万ドルで86-DOSを購入し、IBMにライセンスしたという取引の核心は「売らなかった」ことにある。IBMは規模・ブランド・資本すべてでMicrosoftを圧倒していたが、OSへの関心は薄く、買い取りに固執しなかった。

なぜ転換点か: この契約によりMicrosoftは「Intelの標準CPU（8088）の上で動く唯一のOS」の所有者になった。IBMのPCが成功するほど、IBM互換機が増えるほど、Microsoftの収益が増える構造が確立した。IBMを「最初の顧客」ではなく「流通インフラ」として機能させた点で、ビジネスモデルの設計が天才的だった。

### TP2: 1990年 Windows 3.0の爆発的普及——GUIの標準化

MS-DOSのコマンドライン操作から、グラフィカルインターフェース（GUI）への移行。AppleのMacintoshが1984年にGUIを先行して実装していたが、Windows 3.0はAppleのプレミアム価格と閉鎖性に対し、「すべてのPCで動くGUI」を実現した。

なぜ転換点か: Windows 3.0以降、「PCを買う＝Windowsを使う」という等式が消費者の中で成立した。それまでのDOS時代は複数のOSが競合していたが、Windows 3.0の普及でその選択肢が事実上消えた。アプリ開発者もWindowsに標準化し、「WindowsのアプリがあるからWindowsを使う」正のフィードバックが始動した。

### TP3: 1998年 DOJ独禁法訴訟——「プラットフォーム独占の法的限界」

MicrosoftがIEをWindowsにバンドルし、Netscapeのインストールをメーカーが妨げるよう求めたことが「独占的地位の濫用」として問われた。判事ジャクソンの「分割命令」は最終的に実行されなかったが、Microsoftは2001年の和解でOEM契約の制限（他ブラウザ排除禁止等）を余儀なくされた。

なぜ転換点か: プラットフォーム支配が「OSの隣接市場への拡張」に及ぶと独禁法に抵触するという判例が確立した。この判決はその後のGoogleの検索独占、Amazon・Appleのプラットフォーム問題への規制論議に直接影響を与えており、「デジタルプラットフォームの独禁法」という問題領域を開いた最初の本格的事例となった。

## 反対と執念

IBMとの交渉でゲイツの母メアリー・ゲイツ（IBM役員と面識があった）が紹介の役割を果たしたというエピソードは広く知られる。しかし本質は、ゲイツとアレンが「IBMにとってOSは重要でない」と見抜いたことだ。IBMの文化では、ハードウェアエンジニアがソフトウェアより格上だった。その非対称な価値観を利用して、IBMは気づかないうちに利益の大半をMicrosoftに渡す構造を作った。

DOJ訴訟の最中、ゲイツの証言ビデオが法廷で公開された。「競合を排除することを意図したか」という問いに対し、意味不明に思える回答を繰り返す映像は、Microsoftの企業イメージを決定的に傷つけた。ゲイツは後に「もっと率直に答えればよかった」と語ったが、実態は「意図的にNetscapeを潰した内部メールの証拠が多数あった」ということだった。

## 考察・私見

MicrosoftのOEM戦略が示す最も深い洞察は「流通チャネルをデフォルト化することが、最強の競争障壁である」ということだ。Windowsが優れていたから勝ったのではなく（実際Windows 3.0まではMacより使いにくかった）、「PCを買った瞬間に入っている」というデフォルトポジションが勝ちを決定的にした。

一方でAppleは逆の答えを選んだ——「ハードウェアを自社で作り、OSをクローズドにすることで品質を担保する」。短期的にはMicrosoftの水平展開が市場を席巻したが、長期的にはiPhoneというデバイスでAppleが反転した。どちらが正しかったかという問いに対する答えは「どの市場・どの時代か」によって変わる。

DOJ訴訟の判決（分割命令の破棄）は、Microsoftに有利な形で決着したが、タイミング的には絶妙だった——2001年以降のPC産業はスマートフォンによって根本から変わり始め、Windowsの独占が意味する市場の重要性自体が縮小し始めたからだ。規制が間に合わなかったのか、それとも市場の変化が規制を不要にしたのか——これはデジタル独禁法問題の核心的な問いだ。

## 未解決の問い

- 1980年の交渉でIBMがMS-DOSの所有権を買い取っていたら、PC産業の構造はどう変わっていたか

- DOJ判決（分割命令破棄・和解）がなく、実際にOSと他ソフトが分割されていた場合、その後の産業構造はどう変わっていたか
- MicrosoftのOEM戦略の「水平展開」モデルと、AppleのApp Storeの「垂直統合」モデルは、どちらが長期的に価値を最大化するか
- Microsoftが2000年代にモバイルに出遅れた原因は、Windowsの成功そのもの（Innovator's Dilemmaの事例）か
